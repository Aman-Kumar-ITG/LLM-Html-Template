MODULE 02

RAG Pipeline

Retrieval-Augmented Generation grounds LLMs in real facts — eliminating hallucination through architecture, not prompt tricks.

01
Document Ingestion
Parse PDFs, HTML, DOCX — extract clean text while preserving semantic structure. Parser quality determines retrieval quality.
02
Chunking & Splitting
Split documents into 256–1024 character chunks with 10–15% overlap. Recursive splitting respects semantic boundaries.
03
Embedding Generation
Transform each chunk into a dense vector using the same model you'll use at query time — this choice is architecturally binding.
04
Vector Store Indexing
Store vectors with metadata in Pinecone, Qdrant, or pgvector. ANN indexing (HNSW) enables sub-millisecond search at scale.
05
Query & Retrieve
Embed the user query, search for top-k similar chunks, apply reranking for precision. Over-retrieve by 3x then rerank down.
06
Generate & Cite
Inject retrieved context into the LLM prompt with a strict grounding instruction. Output includes citations to source chunks.

Overlap Is Not Optional
10–15% overlap ensures context continuity at chunk boundaries. Missing overlap = losing information that spans two chunks.
chunk_overlap = chunk_size * 0.12
Semantic Chunking
Advanced approach: embed sentences, detect topic shifts via cosine similarity drop, chunk at semantic boundaries instead of character count.

Token vs Character Count
Character count is a proxy — actual token count varies by language and content type. Use tiktoken for precise LLM-aware splitting.

attention.py 
import torch, torch.nn.functional as F
 
def scaled_dot_product_attention(Q, K, V, mask=None):
    """Multi-head scaled dot-product attention.
    Q, K, V : (batch, heads, seq_len, d_k)
    Returns  : (batch, heads, seq_len, d_v)"""
    d_k = Q.size(-1)
 
    # Raw attention scores — O(n²) per head
    scores = torch.matmul(Q, K.transpose(-2, -1)) / d_k ** 0.5
 
    # Causal mask: decoder can't attend to future tokens
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))
 
    # Softmax → attention weights summing to 1.0
    weights = F.softmax(scores, dim=-1)
 
    # Weighted sum of value vectors
    return torch.matmul(weights, V), weights
 
# GPT-3 dimensions:
# d_model=12288  n_heads=96  d_k=128
# At 4k ctx: attention matrix = 4096×4096×96
# Memory: ~6GB per layer (fp16)
indexing_pipeline.py 
from langchain.document_loaders import PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain.embeddings import OpenAIEmbeddings
from langchain.vectorstores import Pinecone
 
# ── 1. Load ────────────────────────────────
docs = PyPDFLoader("knowledge_base.pdf").load()
 
# ── 2. Chunk ───────────────────────────────
splitter = RecursiveCharacterTextSplitter(
    chunk_size=512,
    chunk_overlap=64,       # ~12% overlap
    separators=["\n\n", "\n", ". ", " "]
)
chunks = splitter.split_documents(docs)
 
# ── 3. Embed ───────────────────────────────
# ⚠ Pin this model — switching = full re-index
embedder = OpenAIEmbeddings(model="text-embedding-3-large")
 
# ── 4. Store ───────────────────────────────
vs = Pinecone.from_documents(
    chunks, embedder,
    index_name="prod-kb",
    namespace="v2"  # versioned namespace
)
print(f"Indexed {len(chunks)} chunks")
query_pipeline.py 
from openai import OpenAI
client = OpenAI()
 
def rag_query(question: str, top_k: int = 5) -> str:
    # 1. Embed — SAME model as indexing!
    q_vec = embedder.embed_query(question)
 
    # 2. Over-retrieve for reranking headroom
    raw = vs.similarity_search_with_score(
        question, k=top_k * 3
    )
 
    # 3. Cross-encoder rerank → precision +35%
    reranked = cohere_rerank(question, raw, top_n=top_k)
 
    # 4. Build cited context string
    ctx = "\n---\n".join([
        f"[SOURCE {i+1}]\n{d.page_content}"
        for i, d in enumerate(reranked)
    ])
 
    # 5. Generate with strict grounding
    return client.chat.completions.create(
        model="gpt-4o", temperature=0.1,
        messages=[
            {"role":"system",
             "content":"Answer ONLY from context. Cite [SOURCE N]."},
            {"role":"user",
             "content":f"Context:\n{ctx}\n\nQ: {question}"},
        ]
    ).choices[0].message.content
